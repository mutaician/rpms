# Remote Patient Monitoring System (RPMS)

A Simple platform for doctors to monitor patients remotely.

## Features

### 1. Doctor Dashboard & Triage
- **Triage Board**: Automatically sorts patient updates by Risk Level (High, Medium, Low).
- **Patient Management**: View patient details, medical history, and active care plans.
- **AI Care Plans**: Generate detailed daily schedules from simple natural language instructions.

### 2. Patient Portal
- **Simple Daily Tasks**: Patients see a simple to-do list generated by AI.
- **Easy Submission**: One-click submission for all daily tasks.
- **AI Analysis**: Submissions are analyzed by AI for risk assessment and structured data extraction.

### 3. Doctor-Patient Communication
- **Exchanges**: Integrated messaging system for follow-ups.
- **Actions**: Quick actions for "Urgent Revisit", "Further Followup", or "Mark as Healed".

## Tech Stack
- **Framework**: Next.js 16 (App Router)
- **Database**: SQLite with Prisma ORM
- **AI**: Genkit with Google Vertex AI (Gemini)
- **Styling**: Tailwind CSS

## Getting Started

1.  **Install Dependencies**:
    ```bash
    bun install
    ```

2.  **Setup Database**:
    ```bash
    bunx prisma db push
    ```

3.  **Run Development Server**:
    ```bash
    bun dev
    ```

4.  **Login**:
    - Go to `/signup` to create a Doctor or Patient account.

## Project Structure
- `app/`: Next.js App Router pages and API routes.
- `app/actions/`: Server Actions for form submissions.
- `genkit/`: AI Flow definitions (Plan Generation, Answer Extraction).
- `prisma/`: Database schema.
